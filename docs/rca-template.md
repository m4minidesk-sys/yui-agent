# docs/rca-template.md — 障害振り返り標準テンプレート

Issue #72 で指摘された「時系列分析と5W1H掘り下げの欠如」に対応するため、
yui-agent プロジェクトの標準障害振り返り（RCA: Root Cause Analysis）テンプレートを定義する。

---

## 使い方

新しい障害振り返りを行う際は、このテンプレートをコピーして
`docs/rca-YYYY-MM-DD-<issue-title>.md` として保存する。

```
cp docs/rca-template.md docs/rca-$(date +%Y-%m-%d)-<タイトル>.md
```

## 障害振り返り: [Issue Title] — [日付]

### 0. 概要

| 項目 | 内容 |
|---|---|
| 障害タイトル | |
| 検出日時 | YYYY-MM-DD HH:MM JST |
| 影響範囲 | |
| 重大度 | 🔴 Critical / 🟡 Warning / 🟢 Info |
| 関連Issue | #N |

---

### 1. タイムライン（時系列）

**必須**: `git log --follow <file>` および `git show <commit>` で実際の日時を確認して記入する。

| 日時 (JST) | イベント | 関係者 | 成果物/コミット |
|---|---|---|---|
| YYYY-MM-DD HH:MM | requirements.md v0.X で AC が定義される | AYA | requirements.md |
| YYYY-MM-DD HH:MM | Kiro 実装コミット | Kiro | `<hash>` |
| YYYY-MM-DD HH:MM | AYA AC突合レビュー | AYA | レビューコメント |
| YYYY-MM-DD HH:MM | 問題が混入（推定） | — | `<hash>` |
| YYYY-MM-DD HH:MM | 問題が検出 | AYA/han | Issue #N |
| YYYY-MM-DD HH:MM | 振り返り開始 | AYA | このドキュメント |

**チェック**: 以下の問いに答えられているか確認する。
- [ ] 最初から問題だったのか、途中で劣化したのか？
- [ ] どのコミット（ハッシュ）で問題が混入したか？
- [ ] skipif/mock等の変化は一括追加か段階的増加か？

---

### 2. 5W1H 分析

各要素を**具体的な git タイムスタンプ・ファイル名・コミットハッシュ**で記述する。
「〜と思われる」「〜かもしれない」は禁止。不明な場合は「未調査」と明記する。

| 5W1H | 内容 | 根拠（git log / コード引用） |
|---|---|---|
| **Who** (誰が) | どの Kiro セッション / AYA レビューが関与したか | `git log --author` |
| **When** (いつ) | 意思決定/混入の正確な日時 | `git show <hash> --format="%ai"` |
| **Where** (どこで) | 影響ファイル・関数名・行番号 | `git blame <file>` |
| **What** (何が) | 現象の正確な記述 | テスト出力・ログ |
| **Why** (なぜ) | 5 Whys で掘り下げ（下記参照） | — |
| **How** (どうやって) | どのプロセスステップで防げたか | — |

---

### 3. 根本原因分析（5 Whys）

**ルール**: 各 Why は前の Why の答えに対して問う。抽象的な分類（「手順不足」等）で止めない。

**問題文**: [具体的な現象を一文で]

1. **Why 1**: なぜ [問題] が起きたか？
   → **答え**: [具体的な事実]

2. **Why 2**: なぜ [Why 1 の答え] だったのか？
   → **答え**: [具体的な事実]

3. **Why 3**: なぜ [Why 2 の答え] だったのか？
   → **答え**: [具体的な事実]

4. **Why 4**: なぜ [Why 3 の答え] だったのか？
   → **答え**: [具体的な事実]

5. **Why 5**: なぜ [Why 4 の答え] だったのか？
   → **答え（根本原因）**: [これ以上分解できない原因]

### 5. 学習事項

- **良かった点**: 今回のプロセスで機能したこと

- **共有事項**: チームに伝えるべき知見
